Programming Projects

Ironroad


Ironroad is a Python project which uses the Irish Rail API to get real-time data about all trains on the network.

As my first Python project, I faced many hurdles, such as parsing the non-standard API and doing typo-correction. I am very proud of the project, and personally use it regularly. Currently, I am fixing edge-cases caused by the API, and adding QoL changes.

Currently, Ironroad is being rewritten in SvelteKit, and will be deployed as a web app.

BitHunt


Bit Hunt is a Treasure Hunt Web App written in Sveltekit and Typescript, with a PostgreSQL database.

It was created for CS++, the Computer Science Society of TU Dublin, which I am chairing for the 2023/24 academic year. It's first production deployment will be the 2023 Freshers' Treasure Hunt, where First-Year students can learn the campus and win prizes.

SIM


SIM (Short for 'Shop Inventory Manager') started as a simple date-checking program for the shop I work in. It was originally written as an ExpressJS app, using 'ETA' as its templating engine.

This summer, I rewrote it in SvelteKit, and added a PostgreSQL database. This allowed a much more flexible and powerful system that was significantly faster and more reliable.

TrackEngine


TrackEngine was my first ever attempt at an API. It was written in NodeJS, and used ExpressJS as its framework.

With the aim of writing Ironroad Web in SvelteKit, I realised that the XML Irish Rail API was not suitable for use in a web app.

TrackEngine was my solution to this problem, it would take the XML data from the Irish Rail API, and convert it to JSON, and do some pre-processing to return easier-to-handle data. You can read the docs here.

ByteStore


ByteStore is a work-in-progress Society Inventory Management System. It is written in SvelteKit and Typescript, with a PostgreSQL database.

It is designed to be a simple, easy-to-use system for managing the inventory of a society, with features such a users, permission, repeated actions (such as charging), login detail stores, and more.

It is currently in development, and will be finished and deployed by CS++ within the 2022/23 academic year.

Algorithms Assignment


For my First Year Algorithms assignment, I was tasked with creating C a program to handle the students in four new courses.

The program had to be able to add students, remove students, and print the students in a course, as well as all of the students in the system. It had to use linked lists, use various sorts to sort the students, and search the students. I added additional features such as a comprehensive test data generator, and a menu system to make it easier to use as well as other small additions to the project requirements to make it more user-friendly.

Theatre Productions

This production was done as part of Dublin Youth Theatre's 'Big Show' summertime performance, and was performed in the Project Arts Centre. One-Way

One-Way and One-Card was a proposed overhaul to the Irish Transport Network, It was created in a group as part of my Global Citizenship module.

CS++ Constitution

CS++ is the Computer Science Society of TU Dublin, and I was elected chairperson for the 2022/23 academic year.

As part of my commitment as chairperson, I undertook the task of writing the society's constitution. The primary goal was to create a robust constitution with clearly defined powers, roles, events and procedures. This constitution also had to be flexible enough to allow for changes in the future due to the nature of the society.

To achieve this, I created a unique office within the Committee called the 'Unenumerated Officer'. Unenumerated offices can be created and removed by the Committee to fill a role that is not covered by the constitution. This exists to be able to select a person to achieve a single task, such as ensuring GDPR compliance, or focusing on a specific element of our tech stack.
It was vital that this role have no official stated powers or duties, as it is designed to be a temporary role, lasting no-more than one semester.

Another goal was to provide a strong template for future constitutions both within CS++ and hopefully other societies.

The constitution will be presented and voted upon at the first General Meeting of the 2022/23 academic year.
